Machine Learning Notes-经验方法总结

Posted on 2018-10-04

经验方法总结

问题引入

  • 假设我们现在正在研究预测房价问题，当我们测试假设函数的时候，我们发现其中存在着很大的误差，那么我们下一步应该如何去debug我们的学习算法呢

  • 我们可以从以下几个角度取考虑如何提升我们的算法

    • 使用更多的训练样例
    • 减少特征数
    • 增加特征数
    • 增加多项式特征
    • 减小λ的值
    • 增加λ的值

  • 但是，并不是每一种方法都是有效的，那么我们该如何知道哪里出了问题呢

模型评估

  • 对于某些问题，我们可以通过MATLAB/Octave来绘制出假设函数的拟合曲线，但是这种方法仅限于某些简单的问题

  • 为了能有效地评估我们的模型，先要将数据集分成两个部分，第一部分为训练集(training set)，第二部分为测试集(test set).主义，在数据集分割的时候，最好先打乱数据的顺序，以免数据集本身的顺序对我们的评估造成影响，通常我们将原数据的70%作为训练集，30%作为测试集

  • 对于线性回归问题，训练/测试的过程如下:
    • 通过训练集求得θ
    • 带入测试集中计算误差(test set error)

  • 那么我们应该使用什么来表示一个模型对未来数据的泛化能力呢？

  • 你也许会想模型在测试集上的误差应该可以表示这个模型的泛化能力，这样真的好么？我们可以看到如下的一个例子

  • 假设我们现在有10个待选择的模型如下图所示，为了选出最好的模型，我们不妨设参数d为模型最高阶的阶数，例如，d=1时它是一个一次式，d=2时它是一个二次式等等.我们按照上一节的测试集的方法，首先在训练集上得到每个模型的参数Θ(1),Θ(2),……,Θ(10)，然后使用这些参数来计算对应模型在测试集上的误差Jtest(Θ(1)),Jtest(Θ(2)),……,Jtest(Θ(10)).通过比较发现，Jtest(Θ(5))最小，所以我们选择这个模型，并且以Jtest(Θ(5))来代表该模型对未知数据的泛化能力.仔细想想，用测试集误差来代表该模型对未知数据的泛化能力是不公平的.因为，这个测试集误差是我们通过对比选择出来的，它在这个测试集上肯定是最优的，相当于我们已经看到了这些数据，用它来代表对未知数据的泛化能力显然不行

偏差与方差

正则化

  • 假设我们的模型是一个高阶多项式，我们想使用正则化来避免过拟合，当λ选择不同值的时候有不同的效果，如下图所示.我们可以把λ看作是正则化的强度，当λ很大时，正则化过强，就会导致模型变得简单即产生偏差(Bias)；当λ很小时，正则化过弱，就可能会导致(Variance).那么问题就在于，我们应该如何选择λ的值？

  • 我们如下定义J(θ)、Jtrain(θ)、Jcv(θ)、Jtest(θ)，注意只有J(θ)带正则化项

  • 我们从0开始，将步长设为2倍速度增长，直到获得一个比较大的值，我们最终取值10.因此，我们得到了12个不同的正则化参数λ对应的12个不同的模型.然后分别训练得到相应的θ，再使用这12个θ计算得到12个Jcv(θ)，然后选择最小的Jcv对应的λ.例如Jcv(θ(5))是最小的，那么我们就选择第5个λ，并计算出Jtest(θ(5))来表示这个模型的泛化能力

  • 下面我们来看看，λ值的变化对Jtrain(θ)和Jcv(θ)的影响.如果λ很小，即几乎没有使用正则化，那么就会出现过拟合；如果λ很大，则会出现欠拟合.这里的图形都是比较理想化的，根据真实的数据所画出的图形可能比这个要乱并且由噪声，但大概都可以看出一个趋势，这样可以帮助你选出验证误差最小的那个点，也就对应了一个理想的λ

总结

方案 可以解决的问题
搜集更多的数据 过拟合
使用更少的特征 过拟合
增加λ的值 过拟合
增加额外的特征 欠拟合
增加多项式的特征 欠拟合
减小λ的值 欠拟合

  • 最后我们再来看看神经网络中的偏差和方差问题.如下图所示，在“较小”的神经网络中，虽然计算资源消耗较少，但是容易出现欠拟合的问题，在“较大”的神经网络中，会消耗比较大的计算资源，也会出现过拟合的问题，但是我们可以使用正则化的方法来解决这个问题，这样的神经网络比“较小”的神经网络要更为有效
