支持向量机

代价函数

  • 我们先来回顾一下逻辑回归，如下图所示

  • 在逻辑回归中，代价函数如下所示，其中hθ为sigmoid函数

  • 我们将hθ替换为sigmoid函数后，当y=1的时候，式子的后半部分为0；当y=0的时候，式子的前半部分为0

  • 如果令-θTx为z的话，我们可以将代价函数看作是关于z的函数.当y=1或y=0时，可以绘制如上图所示的函数图像.在进行一定的近似后，我们就可以得到支持向量机(Support Vector Machine)的代价项

  • 对应的我们称y=0时，支持向量机的代价项为Cost0(z).y=1时，支持向量机的代价项为Cost1(z)

  • 下面我们看一下，支持向量机中的代价函数是什么样的.如下图所示，我们将逻辑回归中的两个部分用Cost0(z)和Cost1(z)替换，并且去掉1/m常数项，最后再去掉λ并加上常数项C，这样我们就得到了支持向量机的代价函数

最大间隔

  • 根据Cost0(z)和Cost1(z)的图像可以知道，在支持向量机中，当y=1的时候，我们希望0Tx≥1而不是0T≥0；当y=0的时候，我们希望0Tx≤-1而不是0Tx≤0

  • 现在假设C是一个非常大的数，例如100,000，我们看看支持向量机会有什么结果

  • 在C非常大的情况下，我们想要最小化代价函数，那么就得有第一项为0.而想要第一项为0，我们需要保证当y=1时0Tx≥1或者当y=0的时候0Tx≤-1.在此约束下，我们的优化问题就变成了

  • 从下图可以看出，在解决上述优化问题时（暂时不考虑如何解决的），我们会发现，支持向量机会选择一个具有最大间隔(margin)的决策边界，所以支持向量机也被称为大间距分离器

  • 当C非常大的时候，支持向量机容易受到异常点的影响.如下图所示，对于该数据集，支持向量机会得到一个非最大间隔的决策边界

  • 如果当正则化参数C不是非常大的时候，最终也会得到理想的最大间隔决策边界，并且即使数据不是线性可分的情况下，支持向量机也可以将其恰当分开

  • C的作用相当于1/λ.当C非常大时，即λ非常小，模型容易出现过拟合，所以一般C不能取得过大
